# Experiments and Results

We evaluated the Legal Document Analyzer using a separate automated evaluation framework that downloads public legal and policy documents, uploads them to the live RAG system, issues a standardized question set, and records response behavior. For the main comparison, we used a corpus of 10 public legal and policy documents and generated 50 evaluation queries, with 5 questions per document spanning factual, extraction, and analytical categories. Each run recorded success rate, number of retrieved source chunks, latency statistics, and model confidence labels.

We first established a baseline configuration using the original retrieval and prompting setup. That run achieved a `100.0%` query success rate across all 50 queries, with an average of `5.0` retrieved source chunks per answer. However, latency was relatively high, with a mean response time of `3998 ms`, a median of `3756 ms`, a P95 latency of `6437 ms`, and a P99 latency of `7606 ms`. Confidence distribution was skewed toward medium-confidence responses, with only `18.0%` of answers marked high confidence and `82.0%` marked medium confidence.

To improve both responsiveness and answer directness, we introduced four optimizations: reducing retrieval depth from 5 chunks to 3, using smaller chunk sizes during ingestion, replacing the long general-purpose prompt with shorter question-type-aware prompts, and evaluating with `gemini-2.5-flash`. The optimized configuration preserved the `100.0%` success rate while materially improving both latency and confidence. Mean latency dropped to `2954 ms`, median latency to `2968 ms`, P95 latency to `3578 ms`, and P99 latency to `3849 ms`. At the same time, the proportion of high-confidence answers increased from `18.0%` to `76.0%`, while medium-confidence answers fell to `24.0%`.

Performance by question type also improved. In the baseline run, factual, extraction, and analytical queries averaged `2607 ms`, `4677 ms`, and `5421 ms`, respectively. Under the optimized configuration, these changed to `2738 ms`, `2978 ms`, and `3340 ms`. Although factual questions became slightly slower, extraction and analytical questions improved substantially, with analytical latency reduced by over 2 seconds on average. This suggests that the new prompting and retrieval strategy is especially effective for more complex legal reasoning tasks.

Overall, the experiments show that the optimized configuration provides a clearly superior operating point for the system. It maintains full reliability, reduces average and tail latency, lowers retrieval overhead, and substantially increases the proportion of high-confidence answers. These results indicate that careful prompt design, reduced retrieval context, and stronger model selection can significantly improve RAG performance in legal document question answering.
